1 · Problem Statement

ร้านจัดการสต็อก 2 สาขาบน Excel + สคริปต์ Python ที่ต้องรันเอง — พนักงานกรอกยาก, มี manual step (copy→paste), path สคริปต์ผูก sandbox รันเครื่องจริงไม่ได้, ไฟล์ xlsx พังง่ายบน Excel Mac. ผล: ข้อมูลไม่ real-time, เสี่ยงกรอกผิด, เจ้าของต้องลงมือเองทุกวัน — ยิ่งขยายสาขายิ่งไม่ scale.

2 · Goals / Non-Goals

Goals

  • พนักงานกรอกสต็อกสิ้นวันจบใน ≤ 5 นาที/สาขา ผ่าน iPad/มือถือ
  • ตัด manual step ทั้งหมด (0 ครั้ง copy/paste หรือรันสคริปต์)
  • ยกมา + คำนวณคงเหลือ + Variance อัตโนมัติ 100%
  • รอบเติม (พุธ/เสาร์) เห็น "ต้องเติมกี่ชิ้นให้ถึง Par" ต่อสาขาทันที
  • เจ้าของเห็นสต็อก + ยอดขายทุกสาขา real-time

Non-Goals (v1)

  • Auto-deduct สต็อกจากยอดขายผ่าน BOM (Phase 2)
  • ระบบสั่งซื้อ/PO ส่งซัพพลายเออร์
  • ตารางพนักงาน / เงินเดือน · POS ตัดเงินหน้าร้าน
  • แอป native (App Store/Play) — v1 เป็นเว็บ responsive

7 · Business Rules

7.1 สูตรคำนวณสต็อก

คงเหลือ (ชิ้น) = ยกมา + รับเข้า − ขาย/ใช้ คงเหลือ (กรัม) = MAX(ยกมา + รับเข้า − ใช้, 0) Variance = ยกมา + รับเข้า − ขาย − ส่งคืน − คงเหลือ (ต้อง = 0) ยกมาวันนี้ = คงเหลือวันก่อนหน้าล่าสุด

7.2 รอบเข้าของแยกวัน/สาขา ⭐

7 รายการ special (Shake แช่แข็ง · Wake up call · Energy Sip · Yellow Madness · Ready to Glow · R-ACAI Bowl · S-ACAI Cup) เข้า สาขาละวันเดียว/สัปดาห์:

7 specialSNDNVP
ตามด้านบนเสาร์พุธ

รายการอื่น (~103) เข้าทั้งพุธ+เสาร์ · ต้องเผื่อ Par ของ 7 special ให้พอถึงรอบถัดไป (7 วัน)

7.3 กรอกเต็ม + เศษ pack (P0-11)

รายการที่ขายเป็นแพ็ค/กล่องแต่เปิดใช้ทีละส่วน กรอก 2 ค่า: จำนวนเต็ม (ที่ยังไม่เปิด) + เศษที่เหลือ (กรัม) — เช่น Greek Yogurt = 1 box + 1500 g; เก็บ 2 หน่วยแยก carry-forward อิสระ

7.4 Reconcile ถ้วยเสิร์ฟ (P0-12)

ถ้วยใช้จริง (ต่อขนาด) = ตั้งต้นวัน + รับเข้า − คงเหลือ Reconcile (ต่อขนาด) = ใช้จริง − ขายจริง(พนง.กรอก) Reconcile (กลุ่มรวม) = Σใช้จริง − Σขาย ← เผื่อสลับขนาด

เช็ก 2 ระดับ: (1) รายขนาด flag ถ้าต่าง · (2) กลุ่มรวม — ถ้ารวมตรงแต่รายขนาดเพี้ยน = mark "น่าจะสลับขนาด" (เตือนเบา)

13.1 Config โหมดขายต่อ item (P0-13)

อ่าน Par Stock UOM (1=แกะ / 2=เต็มกล่อง) + จำนวน/แพค → 46 รายการเป็นแกะ (Greek Yogurt 1kg=1000g, Oreo=454g, Cup=50 ชิ้น) · แก้รายตัวหน้า Settings เก็บลง Supabase

13.2 UOM + Sale Unit / กรอกสองทาง (P0-14, P0-15)

เศษคงเหลือเกินเมื่อวานได้ (แกะกล่องใหม่) แต่ รวม(แพ็ค×N+เศษ) ≤ ของที่มี → เกิน=เตือน · ขาย/ใช้ ↔ คงเหลือ กรอกช่องไหนก็ได้

13.3 กลุ่มเศษรวม (P0-16)

Strawberry/Blueberry หลายขนาด เศษปนกัน → แสดงทุกขนาด, เศษกรอก item เดียว (leader) กันซ้ำ, เช็กยอดรวมกลุ่ม (Σกล่อง×ขนาด+เศษ) ≤ ของเมื่อวาน

13.4 ถ้วย — ดึงจาก stock (P0-12, P0-17)

Cup = 50 ชิ้น/แพ็ค · หน้าสต็อกโชว์ "รวม X ชิ้น" ตลอด · หน้า Reconcile ดึงตั้งต้น/รับเข้า/คงเหลือจาก stock อัตโนมัติ กรอกแค่ขายจริง + จับสลับขนาด

13.5 Par badge + ส่งคืน/เสีย + หมายเหตุ (P0-18, P0-19)

ทุก item โชว์ Par X · ช่อง ส่งคืน/เสีย หักจาก stock ทันที · กรอก>0 เปิดช่องหมายเหตุ (บันทึก returned+note)
